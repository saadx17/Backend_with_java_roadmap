**What is a Variable?**
In programming, a variable is essentially a **labeled storage box** in your computer's memory. It is a way to store data so that your program can remember it, use it, and change it later.
If we think back to the ==JVM architecture== we just explored, when your program is running, it needs a place to keep track of temporary information (like keeping a tally on the **Stack**) or permanent states (stored inside an Object on the **Heap**). Variables are the exact containers doing that work.

Because Java is a **"strongly typed"** language, it is very strict about how you create these boxes. Every variable in Java requires three specific things:

- **Type:** You must declare exactly what _size and shape_ of box you are building. Are you storing a whole number? A decimal? A word?

- **Name:** The unique label you write on the outside of the box so you can find it later.

- **Value:** The actual data you place inside the box.

Here is what that looks like when translated into actual Java syntax:

```
   // Type Name Value
   int age =  22;
   double price = 19.99;
   String dogName = "Rex";
```

**The Golden Rule of Java Variables**
Because you defined `age` as an `int` (an integer/whole number), that box is permanently shaped to only hold whole numbers. If you try to put a word into it later (like `age = "Twenty";`), the Java Compiler (`javac`) will instantly throw an error and refuse to compile your code. This strictness is exactly what makes enterprise Java so reliable, it prevents massive data mismatches from crashing the system while the program is running.
