**History of Java!**
The programming language Java was developed by **James Gosling and his team** at **Sun Microsystems** in the early ==1990s==. Initially the goal was to build a platform independent language for embedded devices such as set-top boxes and televisions.
After exploring C++ (which proved too heavy and too closely tied to platform dependencies), the team embarked on an entirely new project codenamed the *“Green Project”*.
The project first used the name ==Green== (internally “Greentalk” with extension .gt), then was renamed ==Oak== (inspired by an oak tree outside Gosling’s office). When Sun attempted to trademark “Oak,” they discovered it was already claimed by Oak Technologies, so a further rename was needed.
In an interesting session guided by a naming consultant, the team generated dozens of candidate names. The name “Java” (after the Indonesian coffee bean) emerged as the one that cleared legal review. Although Gosling’s preferred name was “Lyric” and the team initially favoured “Silk”, “==Java==” became the official name.

**What is Java?**
Java is a high-level, object-oriented programming language known for its strict syntax, security, and reliability. Its defining philosophy is "Write Once, Run Anywhere" (WORA). This means that once you write and compile Java code, it can run on any device or operating system that has a Java Virtual Machine (JVM), without needing to be rewritten or recompiled for different machines.
In the industry, Java is a heavyweight. It is the backbone of massive enterprise backend systems (like those used in global banking and e-commerce), native Android application development, and large-scale data processing tools.

**What is JDK?**
JDK means **Java Development Kit**. This is the complete toolbox for developers. It includes tools for writing and compiling code, most importantly the compiler (`javac`). The compiler takes your raw Java code (`.java` files) and translates it into an intermediate language called **bytecode** (`.class` files).

**What is JRE?**
JRE means **Java Runtime Environment**. This is the environment required to run a java application. It contains core libraries and the JVM, but it does not contain development tools like the compiler.

**What is JVM?**
JVM means **Java Virtual Machine**. This is the engine inside the JRE. The JVM takes the universal bytecode generated by the compiler and translates it to the fly into machine specific code (0 and 1) for your specific operating system (Windows, Mac, Linux). The JVM is what makes Java's "Write Once, Run Everywhere" philosophy possible.

**Note:** If you are building a new Java program to distribute to everyday users, the everyday users only need to _run_ the program, not build or modify it. Providing just the JRE gives them the JVM and the core libraries they need, without the extra baggage of the development tools found in the JDK.

**Object-Oriented Programming**
**OOP** stands for **Object-Oriented Programming**, which is the heart of Java. It is a way of designing software by modeling it after real world things.

OOP has Two ==core concepts==:
1. Classes (Class)
2. Objects.


*Take those concepts as a car.*
**The Class (The Blueprint):** A Class is a master template. It defines the design for a type of item. It doesn't physically exist on the road; it just describes what properties (state) and actions (behaviors) a car _should_ have.

- _Properties:_ Color, brand, current speed.
- _Behaviors:_ Accelerate, brake, honk the horn.

**The Object (The Instance):** An Object is the actual, "physical" car built from that blueprint. It takes up space (in your computer's memory). You can build many distinct Objects from a single Class.

- _Object 1:_ A red Toyota driving at 60 mph.
- _Object 2:_ A blue Ford parked at 0 mph.

They share the same blueprint (the `Car` class), but they have their own unique, independent states.

**Java OOP** has ==4 pillars==.
1. Encapsulation
2. Inheritance
3. Polymorphism
4. Abstraction


Using our `Dog` blueprint.
**Encapsulation (Data Protection):** This is about protecting an object's internal state. Instead of letting anyone directly change your dog's `hungerLevel` variable, you hide it. You force other parts of the program to use a method like `feed()`, which safely updates the hunger level.

**Inheritance (Family Trees):** This allows a new Class to inherit properties and methods from an existing Class. You could create a `Poodle` blueprint and a `Bulldog` blueprint. Both would automatically inherit the states (`isSleeping`) and actions (`bark()`) from the master `Dog` blueprint, saving you from rewriting code.

**Polymorphism (Many Forms):** This allows objects to share the same action but execute it differently. Both your Poodle and Bulldog inherit the `bark()` method, but the Poodle's bark might return "Yip!" while the Bulldog's returns "Woof!". The single `bark()` command takes many forms.

**Abstraction (Simplifying):** This is hiding the complex inner workings and only showing what is necessary. When you tell a dog to `run()`, you as the user don't need to understand the complex code controlling the dog's muscles and balance. You just trigger the action.
